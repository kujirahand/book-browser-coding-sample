# アイデアバンク

アイデアをたくさん集めるためのアプリです。
ブラウザで選択した部分を、アイデアとしてクリップして保存できます。
保存したアイデアは、AIが自動でタグやタイトルと要約をつけて管理します。

ブラウザを右クリックして、「アイデアバンクに保存」を選択すると、選択したテキストがアイデアとして保存されます。

そして、管理画面で、保存されたアイデアを選ぶと、本文とタグを見やすく確認できます。

## 仕組み

アイデアバンクは、ブラウザの拡張機能とサーバーサイドのAI処理で構成されています。

ブラウザの拡張機能は、ユーザーが選択したテキストをサーバーに送信し、保存されたアイデアを表示する役割を担います。

サーバーサイドでは、受け取ったテキストをAIが解析し、タグやタイトル、要約を生成してデータベース(指定フォルダ以下のテキスト)に保存します。

保存するテキストは、<保存フォルダ>/YYYY/MM-DD-タイトル.mdの形式で保存されます。
LLMが生成した title / summary / tags は、同名の JSON ファイルにも保存されます。

## 実装内容

このディレクトリには、READMEの仕様に基づいた最小実装を用意しています。
サーバーはFlaskで動作し、拡張機能から受け取った選択テキストを解析して、タイトル・要約・タグ付きで保存します。
保存されたデータは、`server/ideas/YYYY/MM-DD-タイトル.md` の形式で管理されます。

管理画面は `http://127.0.0.1:8000/` で開けます。
保存したアイデアの一覧、本文、要約、そして大きく表示されたタグが確認できます。

## セットアップと起動

1. サーバー依存関係をインストールします。

```bash
cd server
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. サーバーを起動します。

```bash
python app.py
```

`title/summary/tags` はLiteLLM経由の1回のLLM問い合わせで生成します。事前に下記の環境変数を設定してください。

```bash
export OPENAI_API_KEY="your_api_key"
# 任意（未指定時は gpt-4o-mini）
export OPENAI_MODEL="gpt-4o-mini"
```

サーバー実装はLiteLLMを利用しているため、OpenAI互換以外のモデルを使う場合はLiteLLMが要求するモデル名や環境変数に合わせて設定してください。

3. Chromeで拡張機能を読み込みます。
`chrome://extensions` を開いて「デベロッパーモード」を有効化し、「パッケージ化されていない拡張機能を読み込む」から `extension/` フォルダを選択してください。

4. 任意のWebページでテキストを選択し、右クリックメニューの「アイデアバンクに保存」を実行します。

## API概要

- `POST /api/ideas`
  - 入力: `{"text": "...", "url": "...", "page_title": "..."}`
  - 動作: アイデアを解析し、Markdownとして保存
- `GET /api/ideas`
  - 動作: 保存済みアイデア一覧を返却
- `GET /api/ideas/<idea_id>`
  - 動作: アイデア詳細を返却
