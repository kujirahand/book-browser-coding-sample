# Wailsでテキストエディタを作成してみよう

Wailsを使って作る、シンプルなメモ帳アプリのサンプルです。
Goでネイティブ機能を実装し、フロントエンドはHTMLとJavaScriptで画面を構成しています。保存ボタンを押すと保存ダイアログを開いてテキストを書き込み、読込ボタンを押すとファイル選択ダイアログからテキストを読み込めます。

## このサンプルで分かること

- WailsでGoのメソッドをフロントエンドから呼び出す方法
- 保存ダイアログと読込ダイアログを使ったファイル操作
- Go側のバックエンドと、JavaScript側のUIの役割分担
- ViteベースのフロントエンドをWailsアプリに組み込む流れ

## このプロジェクト(Wailsで作るテキストエディタ)の主なファイル構成

- [app.go](app.go) - 保存処理と読込処理を持つアプリ本体
- [main.go](main.go) - Wailsアプリの起動設定
- [frontend/src/main.js](frontend/src/main.js) - 画面UIとボタンイベント処理
- [frontend/src/style.css](frontend/src/style.css) - 画面スタイル
- [frontend/src/app.css](frontend/src/app.css) - レイアウト調整用スタイル
- [frontend/wailsjs](frontend/wailsjs) - GoメソッドをJavaScriptから呼ぶための自動生成コード
- [wails.json](wails.json) - Wailsのプロジェクト設定

## 動作の流れ

1. アプリ起動時に [main.go](main.go) でWailsを初期化します。
2. [app.go](app.go) の App 構造体を Wails に bind して、フロントエンドから呼べるようにします。
3. [frontend/src/main.js](frontend/src/main.js) では、Save と Load を import してボタンにイベントを設定します。
4. 保存時は Go 側で保存ダイアログを開き、入力されたテキストをファイルに書き込みます。
5. 読込時は Go 側でファイルを選択し、その内容を文字列としてフロントエンドに返します。

## 実行方法

Wailsをインストールした上で、このディレクトリで次のコマンドを実行します。

```sh
wails dev
```

開発モードでは、フロントエンドの変更が即座に反映されるため、UIを調整しながら動作確認できます。

## 配布用ビルド

配布用のアプリを作成する場合は、次のコマンドを使います。

```sh
wails build
```

## 実装のポイント

### Go側

[app.go](app.go) では、Wailsの runtime パッケージを利用してファイルダイアログを表示しています。

- Save メソッド: 保存先を選択してテキストを書き込む
- Load メソッド: ファイルを選択して内容を読み込む
- startup メソッド: 起動時に context を保持し、ダイアログ表示に使う

### フロントエンド側

[frontend/src/main.js](frontend/src/main.js) では、テキストエリアと2つのボタンだけでシンプルなUIを作っています。

- 保存ボタン: テキストエリアの内容を Save に渡す
- 読込ボタン: Load の戻り値をテキストエリアに反映する
- Goの処理結果やエラーは alert で確認する

## 学習のヒント

このサンプルは、デスクトップアプリでよく使う「ファイルを開く」「保存する」という基本操作を、Wailsで最小構成で実装した例です。
ここから次のような機能を追加すると、実践的なアプリに発展させられます。

- 新規作成時の内容クリア
- 上書き保存
- 文字コードの選択
- 最近使ったファイル一覧
- 保存前の変更有無チェック
